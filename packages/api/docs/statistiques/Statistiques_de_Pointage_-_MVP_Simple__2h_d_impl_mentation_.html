D. PRÉSENCE EFFECTIVE & COUVERTURE (3 valeurs)

Ce que ça montre: Combien de temps l’employé a réellement travaillé

    "total_pause_hours": 0,             // Total heures en pause
    "avg_daily_hours": 0,               // Moyenne par jour travaillé
    "net_work_ratio": 0                 // % temps travail vs temps total
  }
}

Calcul (déjà presque fait):

// Dans la boucle employés existante, ajouter :
const employeeData = {
  employee: await employee.toJSON(responseValue.MINIMAL),
  period_stats: {
    // ... stats existantes
  },
  
  // 🆕 AJOUT ICI
  effective_presence: {
    total_work_hours: parseFloat(totalWorkHours.toFixed(2)),
    total_pause_hours: parseFloat((totalPauseMinutes / 60).toFixed(2)),
    avg_daily_hours: parseFloat(averageWorkHours.toFixed(2)),
    net_work_ratio: presentDays + lateDays > 0 
      ? parseFloat(((totalWorkHours / ((presentDays + lateDays) * 8)) * 100).toFixed(1))
      : 0
  }
};

Calcul:

// Après le calcul de currently_active (ligne ~350), ajouter :

// Compter combien devrait être présent aujourd'hui (work_days_expected > 0)
const expectedToday = employeesData.filter(emp => 
  emp.period_stats.work_days_expected > 0
).length;

// Calculer couverture
const coverageRate = expectedToday > 0 
  ? parseFloat(((currentlyActive / expectedToday) * 100).toFixed(1))
  : 0;

const missingCount = expectedToday - currentlyActive;

// Ajouter au summary
summary: {
  // ... stats existantes
  
  // 🆕 AJOUT ICI
  team_coverage: {
    timestamp: TimezoneConfigUtils.getCurrentTime().toISOString(),
    currently_present: currentlyActive,
    currently_on_pause: currentlyOnPause,
    expected_today: expectedToday,
    coverage_rate: coverageRate,
    missing_count: Math.max(0, missingCount)
  }
}

3️⃣ Analyse durée sessions

Ce que ça montre: Sessions trop courtes ou trop longues

{
  "session_analysis": {
    "total_sessions": 2,
    "avg_duration_hours": 0,
    "abnormal_sessions": [
      {
        "employee_guid": "xxx",
        "date": "2026-02-26",
        "duration_hours": 0,
        "status": "incomplete"        // incomplete, too_short, too_long
      }
    ]
  }
}

Calcul:

// Après la boucle employés, ajouter :

const sessionAnalysis = {
  total_sessions: 0,
  avg_duration_hours: 0,
  abnormal_sessions: []
};

let totalDuration = 0;
let completedSessionsCount = 0;

for (const emp of employeesData) {
  for (const day of emp.daily_details || []) {
    if (day.work_hours !== null && day.work_hours !== undefined) {
      sessionAnalysis.total_sessions++;
      totalDuration += day.work_hours;
      
      if (day.work_hours > 0) {
        completedSessionsCount++;
        
        // Détecter anomalies
        if (day.work_hours < 4) {
          sessionAnalysis.abnormal_sessions.push({
            employee_guid: emp.employee.guid,
            date: day.date,
            duration_hours: day.work_hours,
            status: 'too_short'
          });
        } else if (day.work_hours > 12) {
          sessionAnalysis.abnormal_sessions.push({
            employee_guid: emp.employee.guid,
            date: day.date,
            duration_hours: day.work_hours,
            status: 'too_long'
          });
        }
      }
    } else if (day.clock_in_time && !day.clock_out_time) {
      // Session non fermée
      sessionAnalysis.abnormal_sessions.push({
        employee_guid: emp.employee.guid,
        date: day.date,
        duration_hours: 0,
        status: 'incomplete'
      });
    }
  }
}

sessionAnalysis.avg_duration_hours = completedSessionsCount > 0
  ? parseFloat((totalDuration / completedSessionsCount).toFixed(2))
  : 0;

// Ajouter au summary
summary: {
  // ... stats existantes
  team_coverage: { ... },
  
  // 🆕 AJOUT ICI
  session_analysis: sessionAnalysis
}

Où l’ajouter: Dans l’objet summary (après team_coverage)

E. CONTEXTE & JUSTIFICATIONS (2 valeurs)

4️⃣ Absences justifiées vs non justifiées

Ce que ça montre: Qui a fourni une justification

{
  "justification_status": {
    "total_absences": 3,
    "with_memo": 0,                     // Ont créé un mémo
    "without_memo": 3,                  // Pas de mémo
    "pending_validation": 0,            // Mémos en attente
    "approved": 0,                      // Mémos approuvés
    "rejected": 0                       // Mémos rejetés
  }
}

Calcul:

// Après la boucle employés, ajouter :

const justificationStatus = {
  total_absences: totalAbsences,
  with_memo: 0,
  without_memo: 0,
  pending_validation: 0,
  approved: 0,
  rejected: 0
};

// Récupérer tous les mémos de la période
const periodMemos = await Memos._list({
  target_user: { [Op.in]: teamMembers },
  incident_datetime: { [Op.between]: [startOfPeriod, endOfPeriod] },
  memo_type: MemoType.ABSENCE_JUSTIFICATION
});

if (periodMemos) {
  justificationStatus.with_memo = periodMemos.length;
  
  for (const memo of periodMemos) {
    const status = memo.getMemoStatus();
    if (status === MemoStatus.PENDING) {
      justificationStatus.pending_validation++;
    } else if (status === MemoStatus.APPROVED) {
      justificationStatus.approved++;
    } else if (status === MemoStatus.REJECTED) {
      justificationStatus.rejected++;
    }
  }
}

justificationStatus.without_memo = totalAbsences - justificationStatus.with_memo;

// Ajouter au summary
summary: {
  // ... stats existantes
  team_coverage: { ... },
  session_analysis: { ... },
  
  // 🆕 AJOUT ICI
  justification_status: justificationStatus
}

Où l’ajouter: Dans l’objet summary (après session_analysis)

5️⃣ Conformité aux horaires attendus

Ce que ça montre: Qui arrive à l’heure selon son horaire assigné

{
  "schedule_compliance": {
    "total_clocked": 2,                 // Employés ayant pointé
    "on_time": 0,                       // Dans la tolérance
    "late": 2,                          // Hors tolérance
    "on_time_rate": 0,                  // % à l'heure
    "avg_deviation_minutes": 97         // Écart moyen vs horaire
  }
}

Calcul:

// Dans la boucle employés, enrichir daily_details avec info horaire

for (const [dateKey, dayData] of dailyEmployeeData.entries()) {
  const employeeDayData = dayData.get(userId);
  if (!employeeDayData) continue;

  const { status, delay_minutes, work_hours, expected_time, ...rest } = employeeDayData;

  // 🆕 AJOUT : Calculer conformité
  let is_within_tolerance = null;
  if (status === 'present' || status === 'late') {
    // expectedSchedule est déjà disponible dans la boucle
    const tolerance = expectedSchedule?.expected_blocks[0]?.tolerance || 0;
    is_within_tolerance = delay_minutes ? delay_minutes <= tolerance : true;
  }

  dailyDetails.push({
    date: dateKey,
    status,
    ...rest,
    delay_minutes,
    work_hours,
    is_within_tolerance  // 🆕 AJOUT
  });
}

// Après la boucle employés, calculer compliance global
const scheduleCompliance = {
  total_clocked: 0,
  on_time: 0,
  late: 0,
  on_time_rate: 0,
  avg_deviation_minutes: 0
};

let totalDeviation = 0;
let deviationCount = 0;

for (const emp of employeesData) {
  for (const day of emp.daily_details || []) {
    if (day.status === 'present' || day.status === 'late') {
      scheduleCompliance.total_clocked++;
      
      if (day.is_within_tolerance === true) {
        scheduleCompliance.on_time++;
      } else if (day.is_within_tolerance === false) {
        scheduleCompliance.late++;
      }
      
      if (day.delay_minutes > 0) {
        totalDeviation += day.delay_minutes;
        deviationCount++;
      }
    }
  }
}

scheduleCompliance.on_time_rate = scheduleCompliance.total_clocked > 0
  ? parseFloat(((scheduleCompliance.on_time / scheduleCompliance.total_clocked) * 100).toFixed(1))
  : 0;

scheduleCompliance.avg_deviation_minutes = deviationCount > 0
  ? parseFloat((totalDeviation / deviationCount).toFixed(1))
  : 0;

// Ajouter au summary
summary: {
  // ... stats existantes
  team_coverage: { ... },
  session_analysis: { ... },
  justification_status: { ... },
  
  // 🆕 AJOUT ICI
  schedule_compliance: scheduleCompliance
}

Où l’ajouter: Dans l’objet summary (après justification_status)

💡 NOTES IMPORTANTES

Ce qui est SIMPLE ✅

  • Toutes les données nécessaires sont déjà collectées
  • Pas besoin de nouvelles queries lourdes
  • Calculs basiques (sommes, moyennes, comptages)
  • Réutilisation des variables existantes

Ce qui est ÉVITÉ ❌

  • Pas de comparaison avec période précédente
  • Pas de calcul d’écart-type ou variance
  • Pas de scoring complexe
  • Pas de prédictions
  • Pas de patterns hebdomadaires
  • Pas de ranking
  • Pas de cache ou jobs async

Performance 🚀

  • Ajout minimal de queries (1 seule pour Memos)
  • Calculs légers (boucles sur données déjà en mémoire)
  • Pas d’impact significatif sur temps de réponse
  • Fonctionne même avec 100+ employés
