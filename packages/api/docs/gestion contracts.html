Les Entités et Tables de Toké - Expliqué simplement

Tu as raison ! Voyons comment tout ça est organisé dans la base de données. Je vais t’expliquer chaque table et son rôle avec des exemples concrets.

🗂️ Les 4 Tables Principales

1. session_templates (Les Modèles d’Horaires)

C’est quoi ? La bibliothèque de tous les horaires possibles dans ton entreprise.

Exemple de contenu :

id tenant_id name valid_from valid_to definition
1 100 “Bureau Standard” 2024-01-01 2024-12-31 {...}
2 100 “Équipe Matin” 2024-01-01 2024-12-31 {...}
3 100 “Équipe Soir” 2024-01-01 2024-12-31 {...}
4 100 “Week-end” 2024-01-01 2024-12-31 {...}

Le champ “definition” contient :

{
  "Mon": [
    { 
      "work": ["08:00", "12:00"], 
      "pause": null, 
      "tolerance": 15 
    },
    { 
      "work": ["14:00", "18:00"], 
      "pause": null, 
      "tolerance": 15 
    }
  ],
  "Tue": [...],
  "Wed": [...],
  "Thu": [...],
  "Fri": [...],
  "Sat": [],
  "Sun": []
}

Scénario : Tu crées un modèle “Bureau Standard” avec :

  • Lundi à Vendredi : 8h-12h puis 14h-18h
  • Pause : 12h-14h
  • Tolérance : 15 minutes
  • Week-end : repos

➡️ Cette table stocke UNIQUEMENT le modèle, pas qui l’utilise !

2. rotation_groups (Les Groupes de Rotation)

C’est quoi ? Les cycles d’alternance pour les équipes qui tournent.

Exemple de contenu :

id tenant_id name cycle_length cycle_unit cycle_templates start_date
1 100 “Rotation 2×8 Sécurité” 2 “week” [2, 3] 2024-01-01
2 100 “Rotation 3×8 Production” 3 “week” [2, 3, 5] 2024-01-01

Décryptage de la ligne 1 :

  • name : “Rotation 2×8 Sécurité” (nom pour le manager)
  • cycle_length : 2 (le cycle dure 2 unités)
  • cycle_unit : “week” (l’unité est la semaine)
  • cycle_templates : [2, 3] (utilise les templates id=2 et id=3 en alternance)
  • start_date : 2024-01-01 (le cycle démarre le 1er janvier)

Scénario - Rotation 2×8 :

  • Semaine 1 : Template “Équipe Matin” (id=2) → 6h-14h
  • Semaine 2 : Template “Équipe Soir” (id=3) → 14h-22h
  • Semaine 3 : Retour à “Équipe Matin”
  • Et ça continue en boucle

Scénario - Rotation 3×8 :

  • Semaine 1 : Template id=2 (Matin : 6h-14h)
  • Semaine 2 : Template id=3 (Soir : 14h-22h)
  • Semaine 3 : Template id=5 (Nuit : 22h-6h)
  • Semaine 4 : Retour à Matin

➡️ Cette table définit COMMENT tourner, pas QUI tourne !

3. rotation_assignments (Qui est Dans Quelle Rotation)

C’est quoi ? Le lien entre un employé et son groupe de rotation.

Exemple de contenu :

id user_id rotation_group_id offset assigned_at
1 501 1 0 2024-01-01
2 502 1 1 2024-01-01
3 503 2 0 2024-01-01
4 504 2 1 2024-01-01
5 505 2 2 2024-01-01

Décryptage :

Ligne 1 : Ahmed (user_id=501)

  • Est assigné au groupe rotation_group_id=1 (“Rotation 2×8 Sécurité”)
  • Avec un offset=0 (il commence en position 0 du cycle)

Ligne 2 : Sophie (user_id=502)

  • Même groupe (rotation_group_id=1)
  • Avec offset=1 (elle commence décalée d’une position)

Comment ça marche concrètement ?

Rotation 2×8 (cycle_templates = [2, 3]) :

Semaine Position 0 (Ahmed) Position 1 (Sophie)
Semaine 1 Template 2 (Matin) Template 3 (Soir)
Semaine 2 Template 3 (Soir) Template 2 (Matin)
Semaine 3 Template 2 (Matin) Template 3 (Soir)

Rotation 3×8 (cycle_templates = [2, 3, 5]) :

Semaine user_id=503 (offset=0) user_id=504 (offset=1) user_id=505 (offset=2)
Sem. 1 Template 2 (Matin) Template 3 (Soir) Template 5 (Nuit)
Sem. 2 Template 3 (Soir) Template 5 (Nuit) Template 2 (Matin)
Sem. 3 Template 5 (Nuit) Template 2 (Matin) Template 3 (Soir)
Sem. 4 Template 2 (Matin) Template 3 (Soir) Template 5 (Nuit)

➡️ Cette table dit QUI est dans QUELLE rotation et à QUELLE position du cycle

4. schedule_exceptions (Les Exceptions)

C’est quoi ? Les modifications temporaires d’horaires.

Exemple de contenu :

id user_id group_id session_template_id start_date end_date reason
1 NULL 100 10 2024-12-25 2024-12-25 “Noël”
2 501 NULL 4 2024-12-05 2024-12-07 “Formation”
3 502 NULL 2 2024-12-10 2024-12-10 “Remplacement”

Décryptage :

Ligne 1 - Exception de groupe (pour tout le monde) :

  • user_id : NULL (pas pour un utilisateur spécifique)
  • group_id : 100 (pour tout le groupe/entreprise)
  • session_template_id : 10 (template “Fermé” ou “Jour férié”)
  • start_date / end_date : 25 décembre uniquement
  • reason : “Noël”

➡️ Le 25 décembre, TOUT LE MONDE utilise le template id=10

Ligne 2 - Exception individuelle :

  • user_id : 501 (Ahmed seulement)
  • group_id : NULL (pas pour un groupe)
  • session_template_id : 4 (template “Week-end” ou horaires réduits)
  • start_date / end_date : du 5 au 7 décembre
  • reason : “Formation”

➡️ Du 5 au 7 décembre, Ahmed utilise le template id=4 au lieu de son horaire normal

Ligne 3 - Remplacement d’un jour :

  • user_id : 502 (Sophie)
  • session_template_id : 2 (template “Équipe Matin”)
  • Date : 10 décembre uniquement
  • reason : “Remplacement”

➡️ Le 10 décembre, Sophie travaille avec l’horaire “Équipe Matin” au lieu de son horaire habituel

🔄 Exemple Complet : Marie, Lundi à Vendredi 8h-12h puis 14h-18h

Étape 1 : Créer le modèle dans session_templates

Insertion dans session_templates :
├─ id: 1
├─ tenant_id: 100
├─ name: "Bureau Standard Marie"
├─ valid_from: 2024-01-01
├─ valid_to: 2024-12-31
└─ definition: {
     "Mon": [
       {"work": ["08:00", "12:00"], "pause": null, "tolerance": 15},
       {"work": ["14:00", "18:00"], "pause": null, "tolerance": 15}
     ],
     "Tue": [...même chose...],
     "Wed": [...même chose...],
     "Thu": [...même chose...],
     "Fri": [...même chose...],
     "Sat": [],
     "Sun": []
   }

Étape 2 : Assigner à Marie

2 options :

Option A - Assignation directe (pas dans les tables montrées, mais dans une table user_schedule_assignments) :

user_id: 501 (Marie) → session_template_id: 1

Option B - Via rotation (si Marie fait partie d’une rotation) :

Dans rotation_groups:
id: 5, cycle_templates: [1], cycle_length: 1, cycle_unit: "week"

Dans rotation_assignments:
user_id: 501, rotation_group_id: 5, offset: 0

Étape 3 : Marie travaille normalement

Lundi 4 décembre 2024 :

Toké cherche dans cet ordre :

  1. schedule_exceptions → Aucune exception pour user_id=501 le 4 déc
  2. rotation_assignments → Marie est dans rotation_group_id=5
  3. rotation_groups → Le groupe 5 utilise template id=1
  4. session_templates → Template id=1 = “Bureau Standard Marie”

Résultat : Marie doit travailler 8h-12h puis 14h-18h

🔄 Exemple Complet : Équipe de Sécurité en Rotation 2×8

Configuration complète :

1. Créer les 2 templates dans session_templates :

Template Matin (id=2):
├─ name: "Équipe Matin"
└─ definition: {"Mon-Sun": [{"work": ["06:00", "14:00"], ...}]}

Template Soir (id=3):
├─ name: "Équipe Soir"
└─ definition: {"Mon-Sun": [{"work": ["14:00", "22:00"], ...}]}

2. Créer le groupe de rotation dans rotation_groups :

id: 1
├─ name: "Rotation 2×8 Sécurité"
├─ cycle_length: 2
├─ cycle_unit: "week"
├─ cycle_templates: [2, 3]  ← alternance entre template 2 et 3
└─ start_date: 2024-01-01

3. Assigner les employés dans rotation_assignments :

Ahmed (user_id=501):  rotation_group_id=1, offset=0
Sophie (user_id=502): rotation_group_id=1, offset=1

4. Le système calcule automatiquement :

Semaine du 4 décembre (semaine 49 de 2024) :

Calcul pour Ahmed (offset=0) :

Semaines depuis start_date: 48
Position dans cycle: 48 % 2 = 0
Offset appliqué: (0 + 0) % 2 = 0
Template utilisé: cycle_templates[0] = id=2 (Matin)

Calcul pour Sophie (offset=1) :

Position dans cycle: 48 % 2 = 0
Offset appliqué: (0 + 1) % 2 = 1
Template utilisé: cycle_templates[1] = id=3 (Soir)

Résultat semaine 49 :

  • Ahmed → 6h-14h (Matin)
  • Sophie → 14h-22h (Soir)

Semaine du 11 décembre (semaine 50) :

  • Ahmed → 14h-22h (Soir)
  • Sophie → 6h-14h (Matin)

🚨 Exemple avec Exception

Scénario : Sophie doit remplacer un collègue le 10 décembre

1. Situation normale (sans exception) :

Sophie → rotation_assignments → rotation_group_id=1 → offset=1
Semaine 50 → position 0 → (0+1)%2 = 1 → template id=3 (Soir 14h-22h)

2. Le manager crée une exception dans schedule_exceptions :

id: 10
├─ user_id: 502 (Sophie)
├─ group_id: NULL
├─ session_template_id: 2 (Équipe Matin)
├─ start_date: 2024-12-10
├─ end_date: 2024-12-10
└─ reason: "Remplacement Jean"

3. Le 10 décembre, Toké vérifie dans cet ordre :

1. schedule_exceptions pour user_id=502 le 10/12 ?
   → OUI ! Exception id=10 → template id=2
   → STOP, on utilise ce template
   
Sophie travaille 6h-14h (Matin) au lieu de 14h-22h (Soir)

4. Le 11 décembre (jour suivant) :

1. schedule_exceptions pour user_id=502 le 11/12 ?
   → NON (l'exception était juste le 10)
   
2. rotation_assignments ?
   → OUI → retour à l'horaire de rotation normal
   
Sophie retourne à 14h-22h (Soir)

❓ Questions pour Vérifier la Compréhension

Q1 : Si je veux que Jean travaille 10h-18h tous les jours, quelle(s) table(s) j’utilise ?

Réponse 1. Créer un template dans `session_templates` avec 10h-18h 2. L'assigner à Jean (dans une table d'assignation utilisateur)

Q2 : J’ai 3 équipes qui tournent Matin/Midi/Soir. Combien de lignes dans chaque table ?

Réponse - `session_templates` : 3 lignes (1 pour chaque équipe) - `rotation_groups` : 1 ligne (le groupe qui contient les 3 templates) - `rotation_assignments` : 3+ lignes (1 par employé avec offset 0, 1, 2)

Q3 : Jour férié le 25 déc, personne ne travaille. Quelle table ?

Réponse `schedule_exceptions` avec : - user_id: NULL (tout le monde) - group_id: [id de l'entreprise] - session_template_id: [template "Fermé"] - date: 25/12

Voilà ! Maintenant tu vois comment les données sont organisées et comment Toké les utilise pour calculer automatiquement qui travaille quand ! 🎯
